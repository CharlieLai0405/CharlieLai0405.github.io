第01講 Foundation - 計算機網路概論（黃能富教授）

第01講 Foundation

計算機網路概論 — 黃能富教授

Applications Protocol

  1. URL：Uniform Resource Locator
  2. HTTP：Hyper Text Transfer Protocol
  3. TCP：Transmission Control Protocol

傳出過程：
Domain Name Server（用 URL 查 IP）
→ 3-way handshaking 建立連線
→ 用 HTTP 提出 request 與 acknowledgement
→ tearing down TCP connection

Network Connectivity

Important terminologies：

  1. Link
  2. Nodes
  3. Point-to-point（點對點）
  4. Multiple access（多點）
  5. Switched Network（交換網路）
    • Circuit Switched（電路交換）
    • Packet Switched（封包交換）
  6. Packet, Message
  7. Store-and-forward
  8. Hosts
  9. Switches
  10. Spanning Tree（避免迴路）
  11. Internetwork
  12. Router/Gateway
  13. Host-to-host connectivity
  14. Address
  15. Routing
  16. Unicast / Broadcast / Multicast
  17. LAN / MAN / WAN

Cost-Effective Resource Sharing

How to share a link ?

  1. Multiplexing（多工）→ Switch 1
  2. De-multiplexing（解多工）→ Switch 2

FDM: Frequency Division Multiplexing

Synchronous TDM: Time Division Multiplexing
→ 每個資料 slot 在預定時間發送

FDM/TDM 問題：無法互相借用頻寬，效率低

Statistical Multiplexing（統計式多工）：

  • 根據 flow 傳送資料
  • 排程法：FIFO, Round-Robin, Priority, QoS

Logical Channels

  • 應用程式之間的虛擬溝通路徑
  • 與實體路徑無必然對應

Network Reliability

  • 網路應隱藏錯誤
    • Bit lost / errors / burst errors
    • Packet lost / congestion
    • Link/node failure
    • Delay / out-of-order / eavesdrop

Network Architecture

Protocols

  1. 協定定義相同系統層與對等系統層的介面

  2. 兩種介面：

    • Service interface（縱向）
    • Peer interface（橫向）

  3. 協定規格：pseudo-code / 狀態圖 / 訊息格式

  4. 互通性 interoperable

  5. IETF 為協定標準制定者

  6. 協定架構

Encapsulation

  • 高層訊息被封裝進低層封包
  • 接收端每層只看自己的 header，處理後向上傳遞

OSI Architecture

  • Data Link: 傳一小段
  • Network: 傳整條路徑（如 IP）
  • Transport: 高容錯（如 TCP, UDP）


LAN 通常為 Ethernet（Switch 是 Layer 2）


若含 AP，則為 Ethernet + WiFi

Description of Layers

Physical Layer

  • 傳送 raw bits，媒介如：Coaxial、Twisted pair、Fiber、Air
  • 編碼與 clock 同步為重點

Data Link Layer

  • 將 bits 組成 frame，並傳送至直接相連設備
  • Multiple Access → 需 MAC 解決碰撞
    • CSMA/CD（乙太網路）
    • CSMA/CA（無線）

Network Layer

  • Routing 封包 → 使用 IP
  • Routing protocols：RIP / OSPF / BGP

Transport Layer

  • 提供 process-to-process 通道
  • TCP：可靠
  • UDP：不可靠

Session / Presentation / Application

  • Session：維持應用連線
  • Presentation：格式與編碼
  • Application：HTTP / DNS / FTP / Mail 等

傳輸層與上層僅實作於主機，不出現在 Switch、Router 等設備中

封包名稱比較

  • Layer 2 → frame
  • Layer 3 → packet
  • Layer 4 → message

Internet Architecture

特點：

  1. 非強制分層
  2. Hourglass 架構（IP 為中心）
  3. 新協定需具備：
    • 規格說明書
    • 至少一個實作

Network Performance

  1. Bandwidth：頻寬（bit/sec）
  2. Latency = 傳播時間 + 傳輸時間 + 排隊時間
  • 小檔案 → 傳播時間重要
  • 大檔案 → 頻寬重要

Latency × Bandwidth

表示能「塞入通道」的資料量

  • Latency 是長度
  • Bandwidth 是寬度
  • 設計應達成延遲 × 頻寬的資料利用

Throughput

Throughput = 資料量 / 傳輸時間

  • 傳輸時間 = RTT + 資料量 / 頻寬
  • RTT 對效能影響極大

Performance Summary

  • 效能兩大核心指標：
    • Bandwidth（容量）
    • Delay（延遲）
