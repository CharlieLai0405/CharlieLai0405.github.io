Line Segment Intersection - Map Overlay

計算幾何 Min-Te Sun, Ph.D

Two of Classical Computational Geometry Problems

  1. Line Segment Intersection → 找 rivers and roads 相交的地方
  2. Computing the Overlay of Two Subdivisions

Line Segment Intersection

  • Problem statement:
    給定一組在平面上的 $n$ 條封閉線段，找出它們之間所有交點
  • Time Complexity → Ω($n^2$)

Output-Sensitive Algorithm

  • 當輸出較少時，可節省計算時間
  • 避免暴力檢查所有線段對：
    • 若 x-intervals 沒有重疊，就不可能相交
    • 若 y-intervals 沒有重疊，也不可能相交
  • 以上策略為 Plane Sweep Algorithm（掃描線演算法）
    • 將線段分為「Active Segments」與「Inactive Segments」
    • Observation：交點只會發生在相鄰線段
    • Event Points：所有線段的端點
    • $\ell$ 為「掃描線 Sweep Line」
    • 在掃描時只測試「鄰居」是否相交

演算法示意

HandleEventPoint( p )

  • 定義三個集合：
    • U(p)：以 $p$ 為 upper endpoint 的線段集合
    • L(p)：以 $p$ 為 lower endpoint 的線段集合
    • C(p)：經過 $p$ 的線段集合（但 $p$ 不在端點）

FindNewEvent($S_\ell$, $S_r$, $p$)

Example

  • 若 $p$ 為掃描線上的事件點，則：
    • U(p)：S2
    • L(p)：S4、S5
    • C(p)：S1、S3

Time Complexity

  • $O(n \log n + I \log n)$，其中 $I$ 為所有交點數量
  • 每個操作複雜度為 $O(\log n)$
