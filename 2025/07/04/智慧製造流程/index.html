智慧製造流程

整合設備、自動化、通訊、數據與 AI 的製造架構。

系統資料與控制流程

架構層級與通訊協定說明

1. 設備層（EQP Level）

  • 製程與量測設備，如 AOI。
  • 為原始感測資料的來源。

2. 設備自動化層（EAP Level）

  • 設備控制程式（如 EAP‑1、EAP‑2），常由 PLC 控制。
  • 功能：
    • 上傳感測資料至上層
    • 接收指令並控制設備

3. 自動物料搬運系統（AMHS）

  • 由 MCS（Material Control System）管理搬運與調度。

4. 製造執行系統（MES Level）

  • MES 管理工單、站點流程、即時品質監控（SPC/IPQC）。
  • PMS 與 Recipe 管理。

5. 進階工程系統（AES Layer）

  • 包含：OEE、RMS、APC（R2R）、EDA、DMS、YMS、AMS、PHM。

6. 大數據與 AI 平台

  • 收集跨層資料進行 AI 模型訓練、KPI 預測與視覺化決策。

EAP vs MES 功能比較

功能定位比較

項目 EAP（設備自動化程式） MES（製造執行系統）
層級位置 設備近端（低層控制） 中控層（中階控制）
功能定位 控制參數、監控狀態、回報即時資訊 工單管理、排程、品質監控
通訊協定 SECS/GEM、PLC、OPC-UA 串接 ERP、EAP、REST API
反應速度 毫秒～秒級 秒～分鐘級
對象與範圍 單機控制 整產線或跨區域管理

實際流程分工範例

Step 1 – MES 發出工單

生產 200 片編號 ABC123 的板子，流程為印刷 → AOI → 組裝 → 測試

Step 2 – MES 通知站點（透過 EAP）

  • 傳送配方與批次資訊至 AOI
  • EAP 接收並準備執行

Step 3 – EAP 控制設備動作

  • 拍照、運動控制、燈光調整
  • 結果立即回傳 OK / NG

Step 4 – MES 決策後續

  • NG → 登記追蹤碼並送至重工
  • OK → 自動流轉下一站

MES 與 EAP 的視角差異

面向 EAP MES
設備角度 我該做什麼？怎麼做？ 我今天做誰？為什麼？做到哪裡？
工程師管理任務 控制程式、參數設定、SECS 實作 流程排程、生產查詢、報表分析
異常處理方式 即時警報、設備異常碼、停機回報 分析報告、追溯查詢、維修工單指派

總結

主導層級 主要任務
MES 派工、排程、流程控制、良率追蹤
EAP 控制設備、執行命令、感測器資料回傳

EAP 是即時執行者，MES 是流程指揮官。
EAP 負責「怎麼執行」，MES 決定「做什麼、什麼時候做」。

SECS / GEM 通訊協定介紹

協定簡介

  • SECS（SEMI E5 / SECS-II）：定義設備與主機的訊息格式與流程
  • GEM（SEMI E30）：定義設備狀態、事件、警報與遠端命令的標準模型

協定層級架構

層級 標準 功能說明
傳輸層 SECS-I / HSMS 使用 RS-232 或 TCP/IP 傳輸
訊息層 SECS-II 定義 Stream/Function 訊息結構
功能模型層 GEM (E30) 設備狀態、事件回報、警報與命令功能

常見 SECS/GEM 訊息流程

  • S1F1 / S1F2：主機詢問設備是否在線
  • S2F41 / S2F42：主機下達遠端命令（如開始/停止）
  • S5F1 / S5F2：設備送出警報訊息
  • S6F15 / S6F16 / S6F18：事件回報與註記內容
  • S7F1–S7F4：Recipe 上傳 / 下載

為什麼使用 SECS/GEM？

  • 通訊標準化，可與多設備品牌互通
  • 支援即時製程監控、工單串接、配方控制
  • 有利製程自動化、良率分析與大數據整合

安全風險與建議

  • 協定無內建加密與驗證，風險包括：
    • 中間人攻擊（MITM）
    • Recipe 檔案遭竄改
    • DoS 攻擊（阻斷連線）

建議搭配以下措施：

  • OT 網路隔離、防火牆與安全閘道
  • 權限控管與操作日誌記錄
  • 定期進行資安稽核與白名單策略

總結

  • 智慧製造整合了 EQP、EAP、MES、AES、AI 平台等層級
  • SECS/GEM 提供標準化通訊協定，助於集中管理與製程追溯
  • 搭配 OT 資安架構，可確保資訊透明與製造彈性
